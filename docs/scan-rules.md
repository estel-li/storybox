# 扫描规则

扫描根目录由 `STORY_LIBRARY_ROOT` 配置。

## 目录映射

```text
一级目录 = 分类
二级目录 = 专辑
二级目录以下所有音频文件 = 故事
三级及更深目录 = 章节/分卷路径
```

示例：

```text
01_library_故事库/01_经典名著/三国演义（完结）/001群雄逐鹿/001.英雄问世三结义1.mp3
```

识别为：

- 分类：`01_经典名著`
- 分类显示名：`经典名著`
- 专辑：`三国演义（完结）`
- 章节：`001群雄逐鹿`
- 故事标题：`英雄问世三结义1`
- 排序：`001`

## 音频格式

支持：

```text
mp3 m4a aac wav flac ogg
```

其他文件会被计入总文件数，但不会作为故事入库。

## 排序解析

支持：

```text
001.标题.mp3
001 - 标题.mp3
001 标题.mp3
001_标题.mp3
01.标题.mp3
第001集 标题.mp3
000.引子.mp3
```

无法解析数字前缀时，`sort_order = 999999`。

## 去重和 missing

- 使用 `relative_path` 作为唯一键。
- 重复扫描不会重复入库。
- 重新发现的故事会标记为 `is_missing = false`。
- 本次未发现但数据库中存在的故事会标记为 `is_missing = true`。
